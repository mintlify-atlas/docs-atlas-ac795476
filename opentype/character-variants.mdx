Iosevka's character variant system lets you fine-tune the shape of specific characters without changing anything else. Each variant is exposed as an OpenType feature tag in the `cv##` range (`cv01` through `cv99`). You assign a numeric value to the tag to select a particular shape — for example, `cv36 = 11` selects a single-storey `a` with no serif at the terminal. The result is glyph-level control that you can apply directly in CSS, editor settings, or a custom build plan.

Character variants are available in the **monospace subfamilies** only. The number of variants per character ranges from 2 to over 50 for some glyphs (zero alone has 54 variants).

<Warning>
  Some software limits the number of OpenType features it can process. If you specify many `cv##` tags simultaneously, some may be silently dropped. Validate your configuration in the editor or browser you intend to use.
</Warning>

## How to apply a character variant

<Steps>
  <Step title="Find the cv## tag for your character">
    Each character or character group is assigned a specific `cv##` tag. Consult the table below or the [full variant list in the source repository](https://github.com/be5invis/Iosevka/blob/main/doc/character-variants.md) to find the tag and the index number for the shape you want.
  </Step>
  <Step title="Set the feature in CSS or your editor">
    In CSS, pass the tag with the variant index as the value:

    ```css
    code, pre {
      font-family: "Iosevka Web", monospace;
      /* single-storey a (cv36 = 11), serifed i (cv44 = 5) */
      font-feature-settings: "cv36" 11, "cv44" 5;
    }
    ```

    In VS Code:

    ```json
    "editor.fontLigatures": "'cv36' 11, 'cv44' 5"
    ```
  </Step>
  <Step title="Combine with ligatures and stylistic sets">
    Character variant settings stack with other OpenType features. You can enable `calt` for ligatures, apply a stylistic set as a base, and then override specific glyphs with `cv##` tags:

    ```css
    font-feature-settings: "calt" 1, "ss14" 1, "cv36" 11;
    ```
  </Step>
</Steps>

## Digit form

The `digit-form` selector controls whether digits are lining (all the same height, sitting on the baseline) or old-style (varying heights, some descending below the baseline). This is configured differently from regular `cv##` tags in a custom build plan but affects numeric readability significantly.

| Variant name | Description |
|---|---|
| `lining` | All digits sit on the baseline at cap height (default) |
| `old-style` | Digits have varying heights; some descend below the baseline |

## Using character variants in custom builds

In a `private-build-plans.toml` build plan, character variant assignments go under the `variants` subsection using named keys rather than numeric `cv##` indexes:

```toml
[buildPlans.MyIosevka.variants.design]
a = 'single-storey-serifless'
g = 'double-storey'
i = 'serifed'
l = 'tailed'
zero = 'slashed'
one = 'no-base'
```

Slope-specific overrides use the `upright`, `italic`, and `oblique` subsections:

```toml
[buildPlans.MyIosevka.variants.italic]
a = 'single-storey-tailed'
i = 'tailed-serifed'
```

Assignments under `design` apply to all slopes, and slope-specific entries override them for the corresponding style.
