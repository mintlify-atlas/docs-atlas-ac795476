title: 'Configuring Ligations in Custom Iosevka Builds'
description: 'Configure the calt ligation set by inheriting a language preset, cherry-picking individual ligation groups, and adding PUA compatibility ligatures for Emacs.'
---

The `ligations` subsection controls which programming ligatures are active when an application renders text using the `calt` OpenType feature. You can start from a language-specific preset, then fine-tune individual ligation groups to enable or disable them, or build the set entirely from individual group selections.

## Subsection structure

Ligation configuration lives under `[buildPlans.<PlanName>.ligations]`:

```toml
[buildPlans.IosevkaCustom.ligations]
inherits = "default-calt"   # Optional; inherits an existing ligation set
disables = []               # Optional; disable specific ligation groups, overrides inherited set
enables  = []               # Optional; enable specific ligation groups, overrides inherited set
```

## inherits

**Type:** string — optional

Inherits a predefined ligation set as the starting point. When absent the ligation set starts empty. You can then add to it using `enables`.

## Compatibility ligatures for Emacs

Some applications — notably Emacs — do not support OpenType `calt` ligatures but can render ligatures placed at Private Use Area (PUA) codepoints. The `compatibilityLigatures` subsection lets you map specific character sequences to PUA codepoints.

```toml
[[buildPlans.IosevkaCustom.compatibilityLigatures]]
unicode = 57600 # 0xE100
featureTag = 'calt'
sequence = '<*>'
```

Each entry is an array item (note the double brackets `[[...]]`) with three required fields:

| Field | Type | Description |
|-------|------|-------------|
| `unicode` | integer | PUA codepoint to assign (decimal or hex) |
| `featureTag` | string | OpenType feature to derive the ligature from (`calt` in most cases) |
| `sequence` | string | The character sequence to ligate |

You can add as many entries as needed:

```toml
[[buildPlans.IosevkaCustom.compatibilityLigatures]]
unicode = 57600 # 0xE100
featureTag = 'calt'
sequence = '<*>'

[[buildPlans.IosevkaCustom.compatibilityLigatures]]
unicode = 57601 # 0xE101
featureTag = 'calt'
sequence = '<*<'
```

<Note>
  Remove the `compatibilityLigatures` section entirely when you do not need it. Including it generates additional glyphs at PUA codepoints that may interfere with other fonts in a fallback chain.
</Note>

## Full example from the sample file

```toml
[buildPlans.IosevkaCustom.ligations]
inherits = "default-calt"   # Optional; inherits an existing ligation set
disables = []               # Optional; disable specific ligation groups, overrides inherited set
enables  = []               # Optional; enable specific ligation groups, overrides inherited set

# Compatibility ligatures for Emacs (remove if not needed)
[[buildPlans.IosevkaCustom.compatibilityLigatures]]
unicode = 57600 # 0xE100
featureTag = 'calt'
sequence = '<*>'
```

A JavaScript-focused build that disables arrow ligatures and enables extra equality ligatures:

```toml
[buildPlans.IosevkaCustom.ligations]
inherits = "javascript"
disables = ["arrow-l-hyphen", "arrow-r-hyphen"]
enables  = ["eqeqeq", "exeqeq"]
```
