---
title: 'Configuring Glyph Variants in Custom Iosevka Builds'
description: 'Use the variants subsection to inherit an ss## preset, assign per-character glyph shapes, and override letterforms for each slope in a custom Iosevka build.'
---

The `variants` subsection controls which glyph shape is used for each character in your custom font. You can start from a named stylistic preset and then override individual characters, or build a configuration from scratch. Variants can also differ across slopes — this is how Iosevka achieves its distinctive cursive italic forms.

## Subsection structure

All variant configuration lives under `[buildPlans.<PlanName>.variants]`. The subsection contains a top-level `inherits` key and up to four nested dictionaries:

```toml
# Optional: inherit an ss## preset as the starting point
[buildPlans.IosevkaCustom.variants]
inherits = "ss01"

# Optional: character overrides applied to all slopes
[buildPlans.IosevkaCustom.variants.design]
g = 'single-storey-serifless'

# Optional: overrides for upright (and oblique) only — takes priority over [design]
[buildPlans.IosevkaCustom.variants.upright]
i = 'zshaped'
l = 'zshaped'

# Optional: overrides for italic only — takes priority over [design]
[buildPlans.IosevkaCustom.variants.italic]
i = 'tailed-serifed'
l = 'tailed-serifed'
```

## inherits

**Type:** string — optional

Inherits a complete stylistic set as the base for your variant choices. Any keys you set in `design`, `upright`, `italic`, or `oblique` override the inherited set on a per-character basis.

## design, upright, italic, and oblique dictionaries

**Type:** dictionary — optional

These four dictionaries assign a variant name to a character key. Rules for how they interact:

- `design` applies to **all slopes**.
- `upright`, `italic`, and `oblique` apply to their respective slopes only and **override** any `design` assignment for the same character.

You can use either the human-readable variant name or the equivalent `cv##` tag number:

```toml
[buildPlans.IosevkaCustom.variants.design]
# Named variant
zero = 'slashed'

# Equivalent using the cv## tag number (cv10 = 2 means "slashed")
cv10 = 2
```

### Digit form

The `digit-form` key controls whether numerals use lining (the default) or old-style proportions. It works in all four slope dictionaries:

```toml
[buildPlans.IosevkaCustom.variants.design]
digit-form = 'old-style'
```

| Value | Description |
|-------|-------------|
| `lining` | Digits sit on the baseline at cap height (default) |
| `old-style` | Digits have ascenders and descenders like lowercase letters |

## Full example from the sample file

This example inherits the Andale Mono style (`ss01`), then overrides a few characters per slope:

```toml
# Optional; Whether to inherit a ss## variant
[buildPlans.IosevkaCustom.variants]
inherits = "ss01"

# Optional; Configure single character's variant
[buildPlans.IosevkaCustom.variants.design]
g = 'single-storey-serifless'

# Optional; Configure single character's variant for Upright and Oblique; Overrides [design]
[buildPlans.IosevkaCustom.variants.upright]
i = 'zshaped'
l = 'zshaped'

# Optional; Configure single character's variant for Italic only; Overrides [design]
[buildPlans.IosevkaCustom.variants.italic]
i = 'tailed-serifed'
l = 'tailed-serifed'
```

</Tip>
