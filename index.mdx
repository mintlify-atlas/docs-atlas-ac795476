Iosevka is an open-source, sans-serif + slab-serif, monospace + quasi-proportional typeface family designed for writing code, using in terminals, and preparing technical documents. With hundreds of stylistic variants, full OpenType ligature support, and a powerful custom build system, Iosevka adapts to your exact preferences.

## What makes Iosevka unique

Iosevka ships with extensive built-in flexibility — no patching or modification required:

- **6 monospace subfamilies** — sans-serif and slab-serif, each in Default, Term, and Fixed spacing
- **2 quasi-proportional subfamilies** — Aile (sans-serif) and Etoile (slab-serif) for document use
- **9 weights** from Thin to Heavy, 2 widths (Normal/Extended), 3 slopes (Upright, Italic, Oblique)
- **18+ stylistic sets** (ss01–ss20) that instantly apply looks inspired by Consolas, Menlo, Fira Mono, JetBrains Mono, and more
- **Hundreds of character variants** (cv01–cv99) for per-glyph customization via OpenType
- **Rich ligation sets** for 18+ programming languages including JavaScript, Haskell, Elm, and F#
- **248 supported languages** covering Latin, Greek, Cyrillic, IPA, and more

## Get started

<Steps>
  <Step title="Download Iosevka">
    Visit the [GitHub Releases page](https://github.com/be5invis/Iosevka/releases) and download the package for your preferred variant. Super TTC packages bundle all weights and styles in a single file.
  </Step>
  <Step title="Install the font">
    On **Windows**: drag font files into the Fonts settings panel. On **macOS**: double-click and click Install. On **Linux**: copy to `~/.local/share/fonts/` and run `fc-cache`.
  </Step>
  <Step title="Enable OpenType features">
    In your editor or terminal, enable the `calt` feature for default ligatures. Use `cv##` and `ss##` feature tags to activate character variants and stylistic sets.
  </Step>
  <Step title="Build a custom variant (optional)">
    Create a `private-build-plans.toml` file and run `npm run build -- contents::YourPlanName` to generate a fully personalized Iosevka font.
  </Step>
</Steps>
